# Поиск аналогов номенклатуры

Веб-приложение для подбора аналогов материалов по наименованию.
Работает на GigaChat (Сбер), бэкенд на Node.js, фронтенд на React.

## Как опубликовать на Render — шаг за шагом

Render — бесплатный хостинг, на который можно залить приложение из GitHub.
После публикации получишь обычную ссылку, которая открывается у всех и везде.

Всего три этапа: ① получить ключ GigaChat → ② залить проект на GitHub → ③ создать сервис на Render.

---

### Этап 1. Получение ключа GigaChat (5 минут)

1. Открой [developers.sber.ru/studio/workspaces](https://developers.sber.ru/studio/workspaces)
2. Войди по СберID (если нет — зарегистрируйся, нужен только номер телефона)
3. Создай новый проект:
   - Нажми **«Создать проект»**
   - Выбери API: **GigaChat API**
   - Тип проекта: **Физическое лицо** (бесплатный тариф, 1 млн токенов в месяц)
4. После создания проекта откроется страница с ключами. Найди и скопируй **«Авторизационные данные»** —
   это длинная строка вида `ZjA3MmNiNDgtZDQwOC00YjBkLT...` (это base64 от Client ID и Client Secret вместе).
5. Сохрани её — понадобится на этапе 3.

> **Важно:** в личном кабинете нужно принять оферту использования API, иначе запросы будут отклоняться.

---

### Этап 2. Загрузка проекта на GitHub (10 минут)

GitHub нужен потому, что Render берёт код именно оттуда.

1. Зарегистрируйся на [github.com](https://github.com) (если нет аккаунта)
2. Создай новый репозиторий:
   - Нажми **«+»** в правом верхнем углу → **New repository**
   - Имя: `analog-search` (или любое другое)
   - **Private** или **Public** — на твой выбор
   - Не добавляй README, .gitignore, license — они уже есть в архиве
   - Нажми **Create repository**
3. На открывшейся странице будут команды — нам нужен раздел
   **«…or push an existing repository from the command line»**.
   Скопируй адрес репозитория, он выглядит как:
   `https://github.com/твой-логин/analog-search.git`

4. Распакуй полученный архив `analog-search.zip` в любую папку на компьютере.
5. Открой терминал в этой папке и выполни команды по очереди
   (замени URL на свой, из шага 3):

   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/твой-логин/analog-search.git
   git push -u origin main
   ```

   GitHub попросит логин и пароль/токен — введи их.

> **Если нет git и не хочешь устанавливать:** можно загрузить файлы через веб-интерфейс GitHub.
> На странице репозитория нажми **«uploading an existing file»**, перетащи туда содержимое распакованной
> папки (без самой папки) и нажми **Commit changes**.

---

### Этап 3. Деплой на Render (5 минут)

1. Зарегистрируйся на [render.com](https://render.com) — можно через GitHub-аккаунт, так быстрее.
2. На главной странице Dashboard нажми **New +** → **Web Service**
3. Выбери **Build and deploy from a Git repository** → **Next**
4. Подключи GitHub-аккаунт (если не подключён) и выбери репозиторий `analog-search`.
5. На странице настроек Render автоматически подхватит `render.yaml` —
   но если он спрашивает вручную, заполни:
   - **Name:** `analog-search` (используется в URL)
   - **Region:** `Frankfurt (EU Central)` — самый близкий к РФ
   - **Branch:** `main`
   - **Runtime:** `Node`
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
   - **Instance Type:** `Free`
6. Промотай вниз до **Environment Variables** и добавь:

   | Key | Value |
   |---|---|
   | `GIGACHAT_AUTH_KEY` | (вставь сюда ключ из этапа 1) |
   | `GIGACHAT_SCOPE` | `GIGACHAT_API_PERS` |
   | `GIGACHAT_MODEL` | `GigaChat` |
   | `NODE_VERSION` | `20` |

7. Нажми **Create Web Service**.
8. Render начнёт сборку. Это займёт 3-5 минут — следи за логом.
   Когда увидишь надпись **«Your service is live 🎉»** — готово.
9. Сверху страницы будет ссылка вида `https://analog-search-xxxx.onrender.com` —
   именно её можно отправлять коллегам.

## Возможные проблемы

### При сборке: «Cannot find module» или ошибка установки пакета
Проверь, что в `package.json` есть свойство `"engines": {"node": ">=20"}`. Если нет — добавь.

### Сайт открывается, но при поиске пишет «Ошибка авторизации GigaChat»
- Проверь, что переменная `GIGACHAT_AUTH_KEY` заполнена и без пробелов
- Убедись, что в личном кабинете GigaChat принята оферта
- Проверь баланс токенов в личном кабинете GigaChat

### «Unable to verify the first certificate» в логах
Это нормально для GigaChat — у Сбера свой корневой сертификат (Минцифры),
который не входит в стандартный CA-bundle. В коде уже отключена строгая проверка,
но если ошибка всё же есть — проверь, что ты не редактировал `analog-engine.ts`.

### Сайт «засыпает» после простоя
На бесплатном плане Render сервис останавливается через 15 минут без активности
и просыпается за ~30 секунд при первом запросе. Чтобы избежать — перейди на платный
план ($7/месяц) или настрой пинг через [cron-job.org](https://cron-job.org).

### Хочу обновить приложение
Любой `git push` в репозиторий автоматически запустит новый деплой на Render.

## Локальный запуск (для разработки)

```bash
npm install
cp .env.example .env
# отредактируй .env, вставь свой GIGACHAT_AUTH_KEY
npm run dev
```

Откроется на http://localhost:5000

## Что делает приложение

В приложении две связанные функции:

### 1. Поиск аналогов в интернете (главная страница)

1. Принимает список материалов (через форму или Excel/CSV)
2. По каждой позиции отправляет запрос в GigaChat: что это за товар, какие характеристики, какие производители на рынке
3. GigaChat возвращает список из 4-6 аналогов с обоснованием
4. Если загружен внутренний справочник — приложение находит близкие позиции в нём
5. Результат экспортируется в Excel (два листа: «Аналоги» и «Характеристики»)

### 2. Анализ списка: дубликаты и аналоги внутри списка (страница «Анализ списка»)

Ищет похожие и взаимозаменяемые позиции ПРЯМО в вашем списке, без выхода в интернет. Поддерживается три режима:

- **Быстрый алгоритм** — без AI, мгновенно. Сравнивает наименования по словам, n-граммам и числовым параметрам. Подходит для списков до 1000 позиций.
- **Гибридный** — алгоритм отбирает кандидатов, GigaChat подтверждает или отклоняет каждую пару с обоснованием. Самый точный режим.
- **Только AI** — GigaChat сам анализирует весь список (для списков до 60 позиций).

Позиции группируются по трём уровням: **Дубликаты** (тот же товар, записанный по-разному), **Аналоги** (взаимозаменяемые), **Заменители** (близкие по назначению). Результат экспортируется в Excel с колонкой «Группа».
